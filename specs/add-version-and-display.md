# feat: 增加內部版本號碼機制與介面顯示

## 1. 背景與目標
目前專案的版號僅存在於 Git Tag 與 CHANGELOG 中，程式碼內部缺乏統一的版號定義，且前端介面也沒有顯示目前的版本，導致使用者難以確認目前執行的程式版本。本任務目標是建立內部的版本號碼機制並反應在 UI 上。

## 2. 技術規則
- 在 `app_v2.py` 定義全域變數 `VERSION`。
- 目前最新版號預計為 `v3.4.2` (接續 `v3.4.1`)。
- 將版號注入 HTML 範本，顯示於頁面標題旁或頁尾。

## 3. 變更清單
- [ ] 在 `app_v2.py` 頂部定義 `VERSION = "3.4.2"`。
- [ ] 修改 `ScalpingHandler` 的 `do_GET` 邏輯（或直接在 `HTML_TEMPLATE` 中替換）。
- [ ] 在前端 HTML 結構中找尋合適位置（如 `<h1>` 標題旁）顯示 `v{VERSION}`。

## 4. 驗證標準
- 啟動伺服器後，網頁介面上能清楚看到「V3.4.2」字樣。
- 檢查源碼確認 `window.APP_VERSION` 已正確注入。
